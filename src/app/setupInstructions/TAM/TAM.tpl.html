TAM File Export Instructions

These directions are necessary for the setup, but don't need to be repeated each time the report is generated. After building the custom report, all you'll need to do is export the report and upload the file.

Preliminary TAM Setup - Avoiding Common Problems

To help you get the "cleanest" and most useful customer list from TAM, there is one very helpful thing you can do...

The most common problem with importing data from TAM is the names that show up in the Engine.  In TAM, names can be stored in more than one location, in more than one format, and this can create complications when importing your customers to the Engine.  So, to help avoid these problems, please make sure your data is as "clean" as possible with respect to names.

3 Tips:

  1. The Revolution Engine uses "First Name", "Last Name" and "Company" fields so you can more accurately customize your marketing.
  2. In the Customer section of TAM is a generic "Name" field.  This field is not very reliable since it sometimes contains a company name (for commercial lines clients) and sometimes contains the name of an individual, or even multiple individuals (for personal lines clients).
  3. In the Contacts section of TAM there are more specific "first name", "last name" and "company" fields.  Since these perfectly match the fields that are used in the Revolution Engine, they are the absolute best way to ensure high quality data in the Revolution Engine, so that you can customize your marketing as effectively as possible.

What if I don't have contact in TAM for all of my customers?

  • The best solution is to "clean up" your TAM listings before running your first import.  Go through every customer record in TAM and make sure each one has at least one Contact associated with it, and that at least one (and only one) contact per customer is marked in TAM as the "Primary Contact".  Also, make sure each contact has accurate data in the First Name, Last Name and Company fields, as well as the other contact fields like email, phone, etc. (make sure the email listed for a contact belongs to that individual and isn't their wife or husband's email, etc.)
  • If you absolutely don't have the time or manpower right now to take the step above, you can still import data from TAM's Customer section instead.  However, please be aware that it will not be as "clean" and you may not be able to customize your marketing in certain ways (For example, without a consistent first name, you cannot address your messages "Dear [First Name],")

Creating a saved report

In this step you will be creating and saving a user-defined report. This will allow you to run the report more quickly on an ongoing basis.

  1. In TAM, click "Reports" on the left sidebar
  2. From the TAM Reports screen, click "Search"
  3. Select the "Report Selection" tab.
    • In the "Report Type" box select "Customer"
    • In the "Report Choices" box select "Policies - Current
    • For output type, select "User-Defined"
  4. Select the "Criteria" tab
    • Set each criteria to "All" (if any criteria does not say all, simply double-click it and clear all special criteria from the selection box on the right side, then save changes and it should reset that criteria to "all" by default.
    • Set Record Listings to "List Each Time Found"
  5. Select the "Description" tab
    • Either leave the description blank or enter a description if you want one (like "Customer Export Report for Revolution Engine")
  6. Select the "Fields/Sort" tab
    • a. If there are already items in the field selection area (the box on the right side) clear them all out
    • b. Select the fields listed below. This is done by clicking a field in the box on the left side, then clicking the "Select" button to pull it over into the selection box on the right.

    -Policy Type

  7. Click "Run" (upper-left corner)
  8. When prompted, name the report "Agency Revolution Customer Export" and continue

Exporting From TAM

In this step you will be running the actual customer report.  This report should be run regularly and imported to the Revolution Engine to ensure your data is up-to-date for campaign purposes.

  1. From the TAM Reports screen, click "User Defined"
    • a. In the "Report Type" box, select "User Defined Reports".
    • b.In the "Report Choices" box, select the Agency Revolution export report (should be named something like "Agency Revolution Cust Export" or "AR Contact Export")
    • In the "Output Type" area, select "Excel Raw Data"
  2. Click the "Run" button (upper-left corner)
  3. When prompted, choose "Send output to an existing file".  If this is the first time you've run the report, enter the file name as "Agency Revolution Customer Export" and click OK.  If you've run the report before, you can select the filename from the list of options, then click OK.
  4. After the report runs, you will be asked whether you want to open the file in Excel.Choose NO (you do not need to open it in Excel)

Your report is now ready to import to the Engine!

Importing Your TAM Report

Once you've completed your export, importing is as easy as coming to this webpage. Just select the Upload TAM File button, drop the Excel file into the importer and complete the wizard.

Where to find your TAM Report

With a standard TAM setup, you will find your report saved on the drive where your TAM data is stored. Every network uses different letters for different drives. The TAM drive may therefore be the L: drive, G: drive, or some other letter. Locate the correct drive, then you will find the report by opening the DOC or DOCS folder, then the SP-SHEET (think "spreadsheet") folder. For example, if your TAM drive is the L: drive, the path to that folder should be L:\Docs\SP-Sheet

If you can't find your saved report...

Run the report again and when asked if you want to open it in Excel select yes. When the file opens, go to "Save as..." and take note of the folder location where it's saving to. You can then close Excel and you should be able to locate the file in that same folder location.
